# 9:16 布局规范

## 基础画布

- 画布：1080 × 1920。
- 内容安全区：左 72、右 72、上 112、下 180。
- 推荐字幕区：`x=72, y=1490, w=936, h=180`。
- 最底部 180px 作为平台 UI 与视觉缓冲区，避免放 CTA、二维码和关键结论。

## 信息层级

1. 栏目标签：30px，短句，不超过 10 个汉字。
2. 主标题：76px / 1.18，最多 2 行，建议每行 8–14 个汉字。
3. 引导语：34px，负责承接逻辑，不抢标题。
4. 强调词：90–96px，只保留 1 个结论性名词或短语。
5. CTA：32px，动词或结论，控制在 8 个汉字以内。
6. 字幕：36px / 1.35，每屏 1–2 行。
7. 页脚提示：30px，用于方法、步骤或栏目口号。

## 人物层

- 默认人物框最小 400 × 560。
- 抠图人物使用 `is-cutout`，底部对齐，允许头部轻微越出人物框，但不得压住标题与强调词。
- 普通视频使用 `object-fit: cover`，面部建议位于人物框上半部 35% 范围。
- 人物明暗与主题反差不足时，优先加轮廓光或投影，不要改原始主题主色。

## 长文案处理顺序

1. 删除无信息量修饰词。
2. 拆成“主标题 + 强调词”。
3. 将主标题从 76px 降至 68px。
4. 仍放不下时换布局，不要将字间距压到 `-0.04em` 以下。

## 布局坐标

精确坐标见 `tokens/layouts.json`。CSS 实现位于 `components/portrait-talk-card.css`。
