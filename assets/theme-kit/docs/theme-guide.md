# 配色使用说明

## 原始色板与语义色

`sourcePalette` 是从方案图中抽取的 4 个基准色，不直接对应所有 UI 状态。实际组件使用 `semantic`：

- `canvas`：整张 9:16 画布的主底色。
- `surface` / `surfaceMuted`：标签、字幕、人物框等容器。
- `textPrimary` / `textSecondary`：主次文字。
- `emphasis`：大字号结论词。
- `accent` / `accentStrong`：线条、边框、局部结构。
- `ctaBackground` / `ctaText`：按钮成对使用，不要拆开混搭。
- `speakerSurface` / `speakerFigure`：人物容器和占位轮廓。

## 内容与主题的匹配

- **深空蓝**：理性、数据、技术与方法论。适合“结论先行”。
- **墨绿金**：经营、复盘、策略与管理。金色只做强调，不大面积铺底。
- **暖灰橙**：培训、经验、日常表达。橙色承担行动感，正文保持炭灰。
- **午夜紫**：趋势、创新与未来话题。避免与大量红色信息同屏。
- **极简黑白**：强观点、知识卡片。核心是留白，不是堆黑线。
- **海盐青**：会员、用户、服务与增长。适合长期栏目和高频更新。
- **大地棕**：商业思考、案例与品牌故事。避免再叠加高饱和冷色。
- **活力青柠**：活动、会员日、上新。青柠做局部提亮，正文用深绿色。

## 不建议的混搭

- 不把不同主题的 `canvas`、`textPrimary`、`ctaBackground` 随意拼接。
- 不在浅色主题上使用白色正文。
- 金色、橙色、青柠色不用于大段小字号正文。
- 同屏强调色不超过 2 个，且一个必须为主题主强调色。
