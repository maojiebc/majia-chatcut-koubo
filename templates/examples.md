# 例句与判例库（模板）

复制本文件,把示例换成你自己内容里的真实判例。判例比规则好用:代理拿到「这样的句子这样切」比拿到抽象描述更少出错。

## 气口分卡判例

**好的分卡**（每张卡可独立理解）:

```
信赖感会立马就消失
你再让它有第二次机会 / 很难
但是到了 / 4、5 月份之后 / 它慢慢又开始恢复
```

**坏的分卡**（词内断裂/孤立闪卡/跨句粘连）:

```
到 / 了 4、5 月份之后      ← 词内断裂
第二次 / 机会              ← 拆碎语义单位
你可能可以                 ← 上句结尾+下句开头粘成一卡
```

## 开头钩子三公式（从既有口述中选句,不是写句）

- **数字承诺**:真实数字 + 结果——「500 家门店的周报 10 分钟看完」
- **模式打断**:反直觉断言、无铺垫直进行动——「别再手动导 Excel 了」
- **提问开局**:制造信息缺口——「你的会员数据为什么越算越乱?」

钩子约 2–2.5 秒说完;钩子承诺必须在成片前 1/3 真实兑现。开头的问候、自我介绍、背景铺垫默认删——信息流成片第一句必须是核心论点、结果或反直觉断言。

## 删前保后判例

```
「观、观远 CLI…」            → 删词头卡壳,保完整词
「这个功能很好用,这个功能
  其实特别好用」              → 删前句(后句是完整重说)
「嗯…然后我们看下一步」       → 「然后」承担顺序功能,保留;只删「嗯」
```

一句话原则:**删声音比删意思安全;错删信息的代价高于漏留口癖。**

## 装饰文案判例

**有信息的装饰**(与口播直接相关):

```
连锁餐饮 AI 实战            ← 真实栏目定位
告警过载 → 业务不信          ← 当前段落的业务逻辑
先补数据,再谈模型            ← 行动结论
```

**无效装饰**(禁止):

```
CHATCUT / 02.1 / 9:16       ← 内部工程标签
案例一 / 案例二              ← 占位伪案例
FIELD NOTES · EP.08          ← 英文模板占位
切片 03、内部版本号、页码
```

## 结尾判例

默认在最后一个完整有效口述句与必要自然气口后直接结束。不加模板片尾、品牌落板、循环回环、额外 CTA。可给主音频约 0.25–0.5 秒短淡出(不吞最后音节)。视觉渐暗只有单一合成层能整体接管时才做。
