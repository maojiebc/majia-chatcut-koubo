# 信息块类型与选用

Motion Graphic / 设计图形按「信息块」组织:每块只承担一个任务、绑定一句具体口播、真实数据可追溯、不遮字幕/脸/关键 UI。没有对应证据信号就不加块——装饰性动效是负资产。

## 十类信息块 × 何时用

| 块 | 进入信号(口播里真实出现) | 禁忌 |
| --- | --- | --- |
| **标题卡** | 开场钩子句、章节切换 | 独占标题页 >1 秒;与第一句口播不同帧起 |
| **大数字** | 数字承诺钩子、结论性数据(「500 家门店」「跳出率 45%」) | 数字无来源;一屏多个大数字互抢 |
| **数字变化** | 前后对比、增长/下降过程(「从 3 天缩到 10 分钟」) | 滚动动画时长超过说这句话的时间 |
| **趋势与对比图** | 真实数据的走势、排名、占比 | 彩虹渐变;重画一个「更好看但失真」的版本——真实录屏/截图优先 |
| **步骤清单** | 「第一步…第二步…」连续操作讲解 | 步骤与口播进度不同步;一次全亮 |
| **关键词强调条** | 短而有力的观点句、转折句 | 与字幕原句逐字重复;一条片超过 3–4 次 |
| **章节导航** | 多段结构的进度提示(「三个部分的第二个」) | 为凑结构感硬加;信息流短片通常不需要 |
| **代码/命令卡** | 真实命令、配置、报错信息 | 用示意代码替代真实可运行内容 |
| **问答卡** | 提问开局的兑现时刻、观众高频疑问 | 自问自答堆砌;问题不是内容里真实存在的 |
| **行动引导** | **仅当**用户明确要求,或素材中已有 CTA 且平台路由决定保留 | 默认不加——成片默认自然结束,这是红线不是偏好 |

## 组合纪律

- 一次全片扫描按「视觉机会组合」选块:优先覆盖最难只靠听觉理解的 3–7 个 beat,控制模板重复,连续三段不用同一种块。
- 每块的进入/退出绑定口播句边界(气口),不按固定秒数轮播。
- 块的样式从当前主题的 playbook 取(`assets/theme-kit/playbooks/<id>.md`),不逐块临时挑色。
- 操作演示、数据结果、产品证据**优先真实录屏/原始截图**,信息块只做补充与归纳,不替代可验证的证据画面。
- 每块出现前检查:字幕带、人脸区、关键 UI 三个保护区都不遮挡。

## 画面任务路由（真实素材 vs 信息块）

先按口播语义把同一概念、动作或证据归成一个小段，再判断这段画面的任务；不要按字幕卡机械地一张字幕换一个画面：

| 口播任务 | 优先画面 | 不应做的事 |
| --- | --- | --- |
| 操作演示、软件步骤 | 当前真实录屏 | 用抽象动画替代可验证的操作 |
| 数据结果、产品证明、报错 | 原始截图、结果页或对应录屏 | 重画一个「更好看」但失真的版本 |
| 讲一张图、表或页面 | 原素材为主，叠加圈选、放大、标注 | 让装饰盖住关键数据 |
| 概念、关系、流程、观点归纳 | 信息块 / 关键词动效 | 没有明确任务地堆卡片和动画 |
| A-roll 跳切 | 有语义关联的 B-roll 或克制转场 | 用无关氛围素材掩盖内容断裂 |

一段只解决一个主要视觉任务：真实素材负责可信度，动效负责解释关系；能用原录屏或原图证明的，不用抽象视觉取代。相同素材再次出现时承担不同任务（先全貌、再局部、最后回看结果）。生成资产的成本契约与降级预案见 `operating-manual.md`。

## 与钩子三公式的对应

- 数字承诺钩子 → 开场用**大数字**或**标题卡**兑现同一个数字
- 模式打断钩子 → **关键词强调条**放大反直觉断言
- 提问开局钩子 → 前 1/3 内用**问答卡**兑现答案(钩子与兑现成对,不放空承诺)

## 块级动效词汇表

块的动效收敛成三轴受限枚举——不在词汇表里的动效不用,主题各自的准许/禁用集见该主题 playbook「动效档」。

**进场**(基线 10–16 帧 @30fps,一律按秒或归一化帧率写,见 `motion-transitions.md` fps 归一化):

| 词 | 何时用 |
| --- | --- |
| 淡入 | 默认;一切块的保底进场 |
| 上移入(淡入+轻上移) | 默认搭配;正文类块首选 |
| 左入/右入 | 方向有语义时才用(前后对比、来路去向);不做装饰性换向 |
| 缩放入 | 强调类块一次性使用(大数字/关键词条);同屏不叠用 |
| 弹入 bounce | **默认禁**——「不弹跳」是全包基线;唯一豁免见活力青柠 playbook,且需用户点头 |

**出场**:默认淡出(8–12 帧),优先随气口静默退场;镜像滑出仅当进场方向有语义;**不做戏剧化出场**——出场抢戏=信息块变装饰。

**循环强调**:

| 词 | 约束 |
| --- | --- |
| 脉冲 pulse | 仅数字/状态类块,全片 ≤1 次 |
| 漂浮 float | 几乎不用——持续漂移与「录屏证据要稳」冲突 |
| 闪烁 flicker | **禁**(观感廉价 + 光敏风险) |
| 打字机 typewriter | 仅文字块,且仅当逐字出现本身有语义(如敲命令);不做默认花活 |

**块 × 动效推荐**:标题卡=淡入+上移;大数字=缩放入或滚动到位(数字滚动时长不得超过说这句话的时间);步骤清单=逐项上移入 stagger(间隔约 0.1s);趋势图=从基线生长;关键词条=缩放入一次;其余默认上移入。同一条片同一种进场重复出现是正常的(一致性>花样),同屏两个块同时入场则错开 3–5 帧。
