## 维护原则

- 单次事故先写案例，不直接升级为通用方法。
- 至少经过重复实例和 E2/E3 证据，再晋升到本目录。
- 方法要写适用范围、反例和失败后的回滚动作。
- 能机器判定的稳定方法，才进一步进入 `rules/`。
