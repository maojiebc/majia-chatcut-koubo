# 过渡动效工程

主讲人窗在全屏 ↔ 圆窗 ↔ 矩形窗之间的连续过渡，以及一切带内部时间编排的 Motion Graphic 的工程契约。全部规则来自 60fps 真机导出的实测教训。

## 审美基准

过渡要「iOS 级灵动」而不是「安卓式生硬」：慢起、顺滑、长尾。技术落点——时长基线 21 帧（0.7s，30fps 时间线）；交叉淡化不是过渡（它只混合两张画面，不产生形状与位置的连续变化）。

## 端点契约（最重要的一条）

- 过渡首帧 = 前态可见画面；过渡末帧 = 后态可见画面；**下一帧的稳定画面与后态同位置、同尺寸、同形状**。正向与反向、每一种状态对（A→B、B→C、C→B、B→A）分别验收，不能用一个方向的成功替代其它方向。
- 端点必须以**当前时间线的真实合成结果**冻结（x/y/裁后可见宽高/crop/焦点/圆角/层级），不能拿未经 crop 的媒体外框直接插值——中间态会出现叶片形、椭圆或人脸比例突变。
- 稳定 PiP 至少提前 5–6 帧铺在过渡层下方；**过渡末帧不得把人物 opacity 降到 0**（下方没铺好就降 0 = 成片闪烁消失的头号根因）。
- 末 2–3 帧不允许半透明交接：动画跑满最后一帧、端点像素级一致后，删掉一切 handoff 淡化，否则出双脸重影。

## 帧数与归一化公式

```text
rawN = round(durationSeconds × fps)
rawN < 2  → 直切处理，不冒充动画
N = rawN；写入后回读实际首末可见帧，以实际 N 为准
归一化进度 = localFrame / max(1, N-1)     ← 除以 N-1 不是 N
```

首帧严格 0、末帧严格 1。除以 N 会让末帧只到 (N-1)/N，下一帧残余跳变。

## 分层缓动

「硬」通常有三重根因：帧数太短、多属性共用一条前冲曲线、布尔阈值硬跳。修法：

- 位移用 `Easing.bezier(0.32, 0, 0.12, 1)`；形变（尺寸/圆角）用 smoothstep；阴影随进度连续淡入，不用布尔阈值。
- 三类属性**各自缓动**，不共用一条曲线（前冲曲线 t=20% 就完成 67%，观感必硬）。
- 「先显著变圆、再完成退位」的观感可让圆角进度略早启动，但仍保持同一连续窗口和相同端点。

## 单一合成所有者

一个状态切换边界只留**一个**视觉过渡所有者。落新方案前先清理该边界已有的交叉溶解、淡入淡出、旧 MG、重叠 opacity 和遗留转场项——「几何动画 + 溶解」双重接管是重影和闪回的常见根因。视觉过渡不得顺手改变主音轨、录屏静音、播放速度、source time 或字幕时序。

## 四档可靠性实现链

按顺序选择能通过验证的最高保真路径；每个新会话先在一个代表边界做能力探针，以当前工具描述为真相源：

**ChatCut 宿主快速路由（2026-07 实测，细则与参数契约见 `chatcut-field-notes.md` 窗口动画节）**：第 1 档无原生关键帧；第 2/3 档因 MG 媒体槽（`video`/`image` 属性）运行时失效直接跳过，不再探针；推荐档 = **满屏 item + 窗口 reframe shader**（`submit_shader` 生成，稳定态 start=end 静态窗、过渡态约 12 帧插值，track-bound per-item 挂载）——它同时实现稳定态三层合成与连续过渡。shader 生成不可用或被拒时才落第 4 档直切，降级仍需用户确认。

1. **完整原生时间线关键帧**——工具明确支持位置/尺寸/裁切遮罩/圆角/easing 完整动画组才用；缺任一关键属性就不要用半套关键帧再靠溶解补洞。
2. **JSX 实时媒体 MG**——`create_motion_graphic_from_code` 创建可编辑过渡资产；起终 rect、圆角按同一缓动进度插值；媒体槽在 item 生命周期内固定不换源。JSX 编译成功 ≠ `<Video>` 精确帧解码稳定。
3. **JSX 接缝定帧 MG**——精确帧持续出现缺人/白块/黑块/条带时，把实时视频换成同一接缝的项目内定帧图；只用于约 0.3–0.6 秒短过渡且没有关键鼠标操作必须连续展示的区间；每个边界用自己的定帧，不跨段复用。
4. **气口处直切**——前三档都不可靠时一帧到位；直切不伪装成动画，如实汇报降级原因。

实测警告：一个短 MG 内同时跑两路全分辨率 `<Video>` + 动态 seek 是高风险组合（单帧白块/缺人的主根因）；几何 MG 方案在 60fps 真机导出下曾出现 300px 级位置瞬跳与双窗鬼影——**MG 档必须真机导出验证，不能纸面通过**。故障时按类别减少解码器/换定帧，不要反复调几何参数碰运气。

## fps 归一化（最贵的坑）

时间线 30fps ≠ 导出 30fps。实测导出可能是 50/60fps，MG 里所有按固定帧号写的时间会快 1.67–2 倍——动画跑一半就退场、人物凭空消失 0.17 秒。规则：

```jsx
const frame = useCurrentFrame();
const { fps } = useVideoConfig();
const seconds = frame / fps;          // 首选：按秒编排
const f30 = frame * 30 / fps;         // 或：归一化后复用 30fps 阈值
```

所有 `interpolate(frame, ...)` 和 `frame < N` 一律切到 `seconds` 或 `f30`，不能只改部分章节。

导出验证同理：`submit_export` 的区间参数按**输出帧域**解释，先换算 `exportFrame = round(timelineFrame × exportFps / timelineFps)`；下载后用 `ffprobe` 对账真实 fps/帧数/首尾内容——文件名写着 transition 不能替代对画面内容的检查。

## 已验收动效的指纹守恒

用户给出已验收参考片时，先把其行为冻结为可比对的动效指纹再动手：稳定状态序列；每个转场的方向、半开区间、时长、实现档位、规范化起终几何、easing 与关键 props。修改前后逐项比较 expected/actual——**静态窗、硬切、溶解或全画幅暗幕不能替代清单中的连续动画**；几何端点看似相同但动画实例消失、方向改变或效果数量归零，仍判回归失败。未另行声明时，几何容差 ≤1 个输出像素、时长容差 ≤1 个时间线帧，且仍需合成像素证据。只沿用背景、字幕色或小圆窗而遗漏人物加强态与切换密度，判为没有按参考执行。

## 装饰层生命周期

- 每个可见的非字幕 item/FX 都归属一个 owner（开场/稳定态/过渡/片尾）；装饰层区间不越出宿主状态区间。
- 框线入场动效与生命周期是两件事：draw 动画只控进入段，**不能用 fadeOut 或超长 item 维持稳定态**；框线 item 的 `[start,end)` 与宿主状态完全相等，进入片尾前一切正文装饰精确结束（片尾孤儿框是高频事故）。
- 片尾渐暗必须由单一最终合成层整体接管；多层各自淡出会透出下层旧画面、字幕悬黑底、逐层鬼影。做不到就只做约 0.35s 音频收束，画面自然结束。

## 验证证据链

- 每个过渡实例按目标帧域扫描过渡窗口；至少保存前态末帧、过渡首帧、约 20%/50%/80%、过渡末帧、后态首帧七个证据帧。同方向另一实例的证据不能代替。
- 批量抽帧网格只用于筛查；可疑帧单独请求精确帧并独立复验 ≥2 次，不靠重复渲染到偶然成功。
- 共享 MG 的代码、属性或媒体策略一改，之前所有方向的画面证据全部失效，重验全部实例。
- 502/超时无画面证据时原样重试同一验证请求，不据此改时间线。
