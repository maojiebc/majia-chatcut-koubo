# AI 口播剪辑经验规则吸收记录 v0.1

## 结论

本轮把输入文档视为候选知识源，而不是直接复制进运行时。现有系统已经覆盖内容真实性、字幕硬门、布局状态、预览审批、可恢复执行、媒体 QA、交付包与反馈治理；真正缺失且适合立即机器化的是“视觉候选为什么能被自动采用”。

本轮新增 Visual Decision Contract v0。它把每个语义段的主任务、候选素材、六维评分、来源与版权证据、选择方式、审批状态、反模式和回退动作写成可验证数据。运行时只允许达到透明门槛且证据齐全的候选进入自动选择。

候选知识源使用公开安全的逻辑标识 `knowledge:ai-talking-head-rules:v0.1.0`。仓库不记录个人知识库路径、素材路径、客户名、创作者身份或原文全文。

## 吸收矩阵

| 文档能力 | 当前状态 | 处理 |
| --- | --- | --- |
| 事实、隐私、字幕和发布红线 | 已有 Rule Registry 与 QA | 复用，不重复造规则 |
| A、B-R、B-L、C1、C2、S 视觉状态 | 已有 Creator OS IR | 复用；P 继续作为正交隐私覆盖层 |
| 预览、确认、失败回退、证据链 | 已有审批与可恢复执行 | 复用 |
| B-roll 六维候选评分 | 缺少机器合同 | 本轮实现 |
| 低分不自动插入 | 缺少运行时门禁 | 本轮实现，低于 7 转人工复核或回退 |
| 生成图不得冒充真实证据 | 原则存在，视觉层未落地 | 本轮实现为阻断规则 |
| 固定间隔切镜、无关 B-roll、句句特效等反模式 | 仅有经验描述 | 本轮结构化并阻断自动执行 |
| 字号、语速、卡片字数、动效帧数 | 内容与场景相关 | 仅保留为 Style Profile 的候选基线，不设全局硬规则 |
| 单一“爆款/优秀”总分 | 不可解释且易过拟合 | 不采用；保留分维度证据和决策原因 |

## 本轮机器合同

每个视觉段必须声明一个 `primaryTask`，例如 evidence、speaker、data 或 concept。隐私处理仍由覆盖层负责，不与主构图状态争夺所有权。

每个候选的总分由以下可审计维度相加，满分 10：

- semanticMatch：0–3
- authenticity：0–2
- timingMatch：0–2
- visualClarity：0–1
- rightsConfidence：0–1
- notRecentlyRepeated：0–1

自动选择同时要求：

1. 总分不低于 7，并且记录总分与分项相符。
2. 候选状态为 eligible。
3. 来源已经核验且存在来源证据。
4. 版权置信为 1 且存在版权证据。
5. 预览审批已经通过。
6. 未检测到已登记反模式。
7. evidence 或 data 主任务不得用生成视觉冒充真实证据。

不能满足自动门槛并不等于素材无价值。系统可把它路由为 `manual-review`，但 `executionAllowed` 必须保持 false，直到产生新的受治理审批结果。

## 刻意不硬编码的经验值

原始经验中的字号、单卡字数、动画帧数、固定语速、固定切镜间隔和固定成片时长都可能作为 profile baseline，但不得跨题材、平台和画幅变成 hard rule。它们后续必须通过 profile 合同、可解释偏差和人工修正数据来治理。

## 后续吸收顺序

1. Style Genome：把色彩、字幕密度、动效克制度、人物占比和 B-roll 偏好变成带版本的 profile，而不是散落常量。
2. Quality Dimensions：分别审计可读性、证据充分度、节奏与视觉一致性，不创建无法解释的单一质量分。
3. Human Edit Diff：记录“AI 建议—人工修正—原因”，进入匿名化建议队列；不得自动改写全局规则。
4. Real Adapter Canary：只有真实 ChatCut 能力、媒体探针和渲染结果在受控样片上验证后，才允许从计划合同升级为真实执行能力。

## 数据与发布边界

- 不把私有知识库路径、原文全文或素材身份写入公开仓库。
- 不因 CLI 返回成功就宣称成片验证完成。
- 不自动上传素材，不自动发布到视频平台。
- 未验证的真实 ChatCut 适配器、媒体探针和渲染能力继续保持 fail-closed。
