# 故障恢复手册

连接、会话、项目状态与转写故障的实测处置（全部来自 2026-07 实战：OAuth 中断多场、持久化实测、批量大生产）。

## 连接与传输

- **HTTP 502 / Transport error**：偶发，原参重试即过；批量操作缩小批量再试（如 display_text 50→40 条）。
- **报错但可能已成功**（实锤：transport 报错而操作已生效）：**写操作报错后先回读状态再决定重发**，防双写。读/验证类操作可直接原参重试。
- **HTTP 403 全线出现** = OAuth 会话失效：按宿主的 MCP 配置命令确认配置后重新 OAuth 登录（如 Codex：`codex mcp get chatcut` → `codex mcp login chatcut`）。
- **工具面固化**：已运行任务的 MCP 工具列表不会热更新——重新登录、重启宿主都救不回当前任务。唯一解法：**新开任务**继续。新任务仍拿不到工具面时走续接手册。

## 续接手册协议

连接不可恢复或必须跨任务续接时，把前置成果写盘为续接手册（实证：一份 600 行级手册跨三场会话续跑成功）。严格分两层：

1. **可复用事实层**：规范决定、色板值、术语表、问题清单、已确认的样式参数、状态表——新会话直接采信。
2. **动态 ID 层**：项目/时间线/item/asset/caption key、活动时间线状态——**必须重读**，不得从手册照抄（key 会失效、活动线会漂移）。

手册存放于项目工作目录，命名带日期与任务名。

## 打开既有项目

标准流（实证：批量打开 4 项目）：

1. `list_projects` 拿全项目清单。
2. 有同名项目时**分别打开并编号区分**，避免漏掉。
3. 内置浏览器先查已开 tab 复用，再逐个打开 editor URL（60s timeout）；**勿并发批量 goto**——会集体超时。
4. `target_project` 后 `read_project` 盘点，不信任何旧会话记忆。

**connected ≠ 持久化**（实证：编辑器实例重开后同一项目时间线 3 item → 0 item，素材池仍在）：任何「已保存」声明必须在重开实例后回读验证；编辑完成后如实告知用户刷新确认。

## 转写故障

- **某素材 `transcribing: error` 且 retry 无效**：不无限重转。绕行：同场录制的另一机位（如共享屏幕轨内嵌同音频）转写作时序主轴（实证：说话人轨挂死，全程用屏幕轨转写完成 11 支切片）。
- 转写卡死判定与 `retry_transcription` 按官方 `transcription` skill。
- 字幕突然变英文 / 挂上译文轨：按 `captions-terminology.md` 故障恢复——切回原文源，禁止在译文结果上继续编辑。
- 长素材转写慢且不复用旧缓存、定向短区间转写可能服务端失败：按已知故障重试或改整段转写；大文件分片上传超时可改一次性直传；可只读复用旧项目云端素材副本避免重传。

## 素材上传

- 上传失败续传：新 `create_session` token + `--asset-id` 指向同一资产续传；网慢降 helper 并发、提超时。
- 高分辨率低码率原件（如 2K 屏录）被 helper 转码反而变大：调 `SKIP_MAX_SOURCE_BITRATE_BPS` 阈值直传原件。
- 注册即返回 assetId 可先摆时间线；字幕等 transcription 完成、帧渲染/导出等 upload 完成（否则 `no source URL` 报错）。

## 上下文压缩后恢复

每次压缩/中断后按序恢复，不信压缩前状态：

1. `manage_timelines` 显式 switch 回目标时间线（活动线可能已漂移）。
2. 重读 profile 与命中分册；**不凭摘要续做画面结构**——先回读状态表再动手（SKILL.md 确认闸门）。
3. 工具 schema 丢失时先只列工具名，再按名取单个 description——不要全量 dump 所有工具（动辄数万 token 被截断）。
4. 回读刚才修改的区间，确认无未验证半成品，再向后推进。

## 杂项实测坑

- 抽帧文件下载名与预期不一致：先 `ls` 列目录再引用，不猜文件名。
- shell 大文件读取截断警告：`sed -n 'a,bp'` 分段读。
- 插件安装：按官方指南使用宿主内置安装路径；装完必须**新开任务**才有工具面。
- 内置浏览器接管失联（DOM/截图全超时）：可能是宿主更新致浏览器运行时路径失效，重新初始化浏览器会话；恢复前不盲操作界面。
- 音频异常结论以源件 ffprobe/试听为准；转写中间文件的异常≠源件损坏（实证：曾误报音损，源件实测完好撤销）。
