# 主题配色系统

包装（字幕底板/标题/标签/观点条/MG）的配色规则与 8 套已实测主题。色值唯一真相源 = `assets/theme-kit/tokens/themes.json`；SVG 底图、1080×1920 布局与可运行 HTML 组件都在 `assets/theme-kit/`（打开 `index.html` 看总览）。

**每套主题自带一份代理 playbook**（`assets/theme-kit/playbooks/<id>.md`）：token 语义与实测档位、版式搭配、信息块偏好、字幕底板硬规则、以及一段可直接嵌进 MG 生成指令的「调用 crib」。标准用法：**选定主题 → 完整读它的 playbook → 生成图形时把 crib 嵌进指令**，不再逐元素查表挑色。

## 用法总则

1. **开工冻结一套主题**，记录 `paletteId + paletteVersion + 明暗模式 + 输出色域`；同一条片不逐元素临时挑色，不跨主题混搭角色。
2. **按角色令牌用色**：`canvas`（大面积底）/ `surface`（卡片、字幕底）/ `ink`（正文字幕）/ `inkMuted`（辅助）/ `primaryFill+onPrimary`（品牌填充+反色字，两个角色分别验收）/ `accent`（低频强调）/ 语义色只在真有语义时用。
3. 大多数口播包装只需要「一组中性色 + 一个主色 + 最多一个强调色」。
4. 用户未指定主题时给 1–2 个候选让用户拍板，不代替用户决定视觉气质。

### 已实测档位限制（按最终合成像素复核过）

- `sea-salt-cyan` 的正文色已在 token v1.1 加深为 `#07554F`（对 canvas 约 7.67:1，达 7:1 目标；旧 `#0B5F58` 仅 6.65:1 已废止）；字幕在录屏等复杂背景上仍配 `surface`（`#F7FFFD`）底板保底。
- 深色三套（deep-space-blue / emerald-gold / midnight-purple）的 `accentStrong` 对 canvas 仅 1.8–2.6:1，**只能做填充底**配 `onPrimary` 反色字，禁止直接当文字色。
- 浅色四套的 `border` 对 canvas 仅 1.6–2.7:1，纯装饰线，不承担有信息意义的分隔。
- 各主题 `accent` 多为大标题级（4.8–6.9:1）：只用于大号强调词与数字高亮，不用于正文和小字号。
- 半透明角色（`surfaceMuted`/`speakerSurface`/`glow`）一律以五类背景（亮、暗、肤色、彩色 UI、高细节）的最终合成像素验收。

## 对比度内部目标

视频会经历缩小、运动、编码和复杂背景，所以比 WCAG 更严：

| 内容 | 目标 | 不达标怎么办 |
|---|---:|---|
| 字幕、正文、小标签、关键数字 | `≥ 7:1` | 改前景/底板/字号结构，不靠加阴影抢救 |
| 大标题、短时大号反色字 | `≥ 4.5:1` | 手机尺寸+实际编码后仍清楚才放行 |
| 有信息意义的边框、图标、数据线 | `≥ 3:1` | 同时增加形状、粗细、标签或纹理 |

执行要点：对比度按**最终合成像素**计算（半透明底、渐变遮罩要在真实画面上重测）；白字不自动安全（亮橙 `#FF6B35` 配白字仅 2.84:1，禁用于字幕；配深色字 5.78:1 只适合大号短文案）；颜色不做区分信息的唯一通道（同时用文字/图标/形状）。

## 色板简报（palette brief）

包装开工前冻结一份 palette brief，模板/MG 复用时随资产记录，之后才能整体换肤：

```yaml
paletteId: sea-salt-cyan      # 主题 id，不把版本号拼进 ID
paletteVersion: 1.1.0
mode: light
outputGamut: rec709-sdr
verifiedPairs:                 # 实测过的前景-背景对
  - [ink, canvas, 7.67]
  - [onPrimary, primaryFill, 8.44]
```

`paletteId`、版本、模式和输出色域是模板契约的一部分；不得在同一条片里逐元素临时挑十六进制值，也不得把浅色模板的颜色直接复制到深色模板。

## 感知均匀色阶与色域

- 从品牌种子色生成深浅层级时，优先在 HCT、OKLCH 或 LCh 里做；不要在 HSL 里等步长拉亮度后假设视觉上也等距。
- 同一色族主要靠明度/色调建立层级，彩度在极亮和极暗端主动收敛，避免高光发荧光、暗部发脏。
- 渐变、热力图和动画补间优先在 OKLCH/LCh 中插值；RGB 直插容易出现灰泥中点或不自然色相漂移。
- P3/HDR 包装转 SDR 时做感知色域映射，不把超出色域的通道逐个硬裁到 0/255；正式 SDR 交付检查实际导出。
- 跨模板匹配的颜色在同一画面并排比较最终合成帧；有工具时以 `ΔE2000 < 2` 为一致参考线——「十六进制相同」不保证经过不同色彩管线后看起来相同。

## 图表与数据色板

| 数据关系 | 色板类型 | 规则 |
|---|---|---|
| 从低到高的数值 | Sequential 顺序 | 单一或相邻色相，明度单调变化 |
| 围绕零点/目标值的正负 | Diverging 发散 | 中点必须有业务含义，不放任意均值 |
| 无顺序的多个类别 | Categorical 分类 | 色相区分为主；直接标注类别，避免只放图例 |
| 周期数据 | Cyclical 周期 | 只有时间环、角度等真正首尾相接时用 |

优先从 D3 Scale Chromatic / ColorBrewer 对应家族选，再映射到当前主题的中性色背景。禁止对普通数值使用彩虹色阶；分类超过约 6 个时优先分组、直标、分面或加形状。灰度预览仍应读出顺序与关键差异；色觉缺陷模拟后结论不能消失。

## 检查清单

- [ ] `paletteId` + 版本 + 模式 + 色域已冻结，实例无散落的任意十六进制值
- [ ] 一组中性色 + 一个主色 + 最多一个强调色；语义色未被当装饰
- [ ] 字幕/正文 7:1、大标题 4.5:1、有意义图形 3:1
- [ ] 白字未压在亮黄/亮橙/浅色底上
- [ ] 半透明叠层按五类背景合成像素验收
- [ ] 灰度预览仍有阅读层级

工具：`assets/theme-kit/scripts/check-contrast.mjs` 可机械校验色对；位图底图不入库，用 `assets/theme-kit/scripts/render-backgrounds.py` 从 SVG 再生。
