# 逐片执行手册

批量切片与逐片精修的执行契约、A-roll 删减增量、验证方法学与验收门禁。SKILL.md 五条第一性原则是本手册的上位法。

## 一片一闭环

一次只处理一个切片，按以下顺序完成后再进下一片。不得同时铺开多片修改。

### 1. 冻结与盘点

- 读 active timeline、所有轨道、字幕源、FX 和素材；记录 `timelineEnd`、`outroStart`、主音轨、录屏轨、人物轨、框线轨。
- 「不动结构」时冻结内容清单（revision、源区间/切点、顺序、时长/速度、字幕时序、主音频映射）；有已验收参考时同时冻结视觉清单（状态序列、每个转场的方向/区间/时长/实现档、MG/FX owner）。
- 列出所有可见 item/FX 的 owner；旧白块、重复人物、未知 MG/FX、跨片尾装饰和测试 probe 先入删除清单。

### 2. 建状态表

逐语义段标记 A/B-R/B-L/C1/C2/S/P 七执行状态（语义路由见 `dual-frame-layouts.md`）。P 是独立不透明 redaction owner，可与叙事状态重叠；人物大矩形与录屏裁切只承担构图。边界落在气口或现有切点。状态表先行是 SKILL.md 的硬闸：无状态表禁止批量 `edit_item`。

### 3. 原子改画面

- 先拆跨态媒体，再 `validateOnly` 校验整批 deletes/adds/updates，通过后原样提交。
- 录屏统一应用 profile 裁切与静音；人物 anchor 唯一可听。
- 框线实例区间必须与宿主状态完全一致；框线入场动画只管进入段（作者基线 `drawFrames=12`、`fadeOut=0`），在宿主区间精确结束。
- 每次修复后回读整条轨道；框线轨区间并集必须等于需要框线的状态区间并集。

### 4. 精修字幕

按 `captions-terminology.md` 执行；先应用 profile 再立刻回读配置，`maxLines≠1` 即停。全量页审计，validator 必跑。

### 5. 音频与气口

- 人物 anchor 从第 0 帧可听；先测 integrated/short-term LUFS、true peak 与噪声底再决定增益。重复轨优先 mute/disable；只有宿主无此能力且已记录兼容证据时才降级到 `-60dB`。片尾 click guard 与自然收束分开命名，不用一个固定 fade 覆盖所有情况。
- 连续片段按 `sourceStart + duration/fps × playbackRate` 对账，防前段静音后段重播。
- 变速可从 1.2–1.3x 试听（保持音高）；先完成口误与停顿清理再整体提速，1.3x 让情绪失真时局部保留 1.15–1.25x，不全片机械统一。
- 导出音频样片实听：开头 2 秒、每个源跳转接缝前后 1–1.5 秒、最后一句。**轨道结构连续≠听感连续**；gain/source 参数正确不算通过，必须有实听或电平证据。
- 机械检查：`ffprobe` 时长对账 + `silencedetect`（-38dB）查静音段 + `volumedetect` 首 0.1/0.5/1.0s 窗有声。

### 6. 收尾验收（门禁汇总）

**结构门**：空窗=0、跨态未拆分 item=0、可闻重复轨=0、孤儿 MG/FX=0、未知 override=0、探针残留=0、未分配 redaction owner 的隐私候选=0、`[outroStart,timelineEnd)` 正文 owner=0。

**字幕门**：validator exit=0、多行=0、超限=0、单字卡=0、`<450ms`=0、未批准 `<800ms`=0、繁体=0、已知术语错=0；硬规则读 `rules/policy.json`，数字/专名纠错必须有证据。

**音频门**：首 0.1s 有声、接缝试听通过、片尾无硬截断、无双声。

**像素门**：首帧、每个状态边界 `(N,N+1)` 帧对、过渡 20%/50%/80%、隐私窗口全程、`outroStart−1/0/+1`、最后一帧。无法读取像素时只报 `structure-pass / pixels-unverified`。

**八道硬闸**（历史事故的回归闸门）：

1. **片尾终止审计**：`outroStart` 起除收束 owner 外一切正文装饰必须已结束；逐帧查 `T−1`、`T`、收束中点、最后可见帧。
2. **第二次出现即事故**：同一可见故障本任务第二次出现，停局部补丁，定位共享资产根因，修复后旧验证全部作废、全实例回归。
3. **动效行为守恒**：修改已验收参考风格前先建可见行为清单（状态序列/每类形状转换/进入退出）；不得以静态窗、硬切或暗幕替代已存在的连续动画，除非用户明确批准降级。
4. **首屏暗幕门禁**：第 0 帧至首句落稳，任何顶层包装不得降低脸部亮度或遮人物；需要暗幕/标题页须作为独立视觉段落经用户确认。
5. **开场实听证据**：0–2 秒必须实际试听或电平检查；只看 gain/source 参数不算通过。
6. **二维码全时间线扫描**：先全片扫描发现所有候选窗口，再逐 geometry/state 覆盖验证；不能用某一稳定态的通过顶替过渡窗口。
7. **参考状态表带效果栈**：记录参考片的可见动效类型、持续、进入/退出方式与保护状态，防止只抄到配色。
8. **全量回归矩阵**：共享 MG/模板/属性一改，所有引用切片各验至少一个相关精确帧，任一失败阻断扩散。

## A-roll 内容（官方之上的增量）

官方 `talking-head-guide` 已覆盖口癖两分法、retake 处理、停顿阈值与 Script 工作流，直接执行官方。以下是增量：

- **一句话原则：删声音比删意思安全；错删信息的代价高于漏留口癖。**
- 删前保后：句间重说删前版、句内重启只删错误片段、词头卡壳删第一个不完整词头、多次重说保留最后完整版。有证据或用户指定才反向。
- 风险分级：低（明确空白、纯卡壳、逐字子集）小批处理后抽检；中（单个口头词、短句内重启、0.2–1s 停顿）听前后语境；高（整句、长片段、改写式重说、跑题、专业词附近）逐项核对保留版覆盖独有信息，没证据就保留或交用户。
- 口播稿只做语义对齐与补漏，不逐字 diff，不用稿件替换说话人未说过的内容。
- ASR 头尾盲区：完成删减后检查媒体真实开头结尾（开机动作、收尾杂音、伸手关录制）。
- 候选删除清单≠实际落刀：回读最终时间线并试听实际切点。
- 笑声、反应、结论后的留白、录屏操作停顿可能是内容，不当普通空白清除。
- 信息流切片开头问候/自我介绍/铺垫默认删（预判定见 `retention-structure.md`）。

## 批量流水线

1. **切片指令契约**：锁定三要素 = 数量 + 单片时长档 + 目标平台（画幅/字幕位置/时长按 `retention-structure.md` 平台路由）。
2. **源级资产单次化**：源转写、术语表、素材盘点全批次共享；逐片重转写视为违规。
3. **选段人工闸门**：AI 选段在 transcript 面板逐段确认后才进下游。
4. **模板层冻结**：字幕样式、色板、包装模板在样片确认后冻结；共性错误在模板层修一次全批生效。**但模板层只冻结共性——动效、气口、字幕必须逐片执行验收，逐片精修优先于批量模板化。**
5. **代表片先行**：至少三类——R1 开场、R2 最复杂合成（录屏+人物+隐私/框线）、R3 最复杂片尾；代表片通过只证明模板可扩散，不豁免逐片门禁。
6. **回归矩阵**：每受影响切片一行，记录开场 0–2s 实听、状态/动画清单对账、全部边界、隐私窗口、`outroStart` 交接、orphan/probe/未知 override 计数。每个 `pass` 绑定 timeline revision、帧域、准确帧和证据 ID；revision 或共享实现变化后全部转 `stale`。
7. **重复缺陷计数**：按 `failureSignature + assetVersion + verificationEpoch` 统计；第二次出现走硬闸 2。用户指出的缺陷存为命名 canary（原始时间码+语义边界+失败签名），重定时后按语义边界重定位。

### 多代理调度编制（宿主支持 spawn 时）

- **分片包干**：批量审计按 2–4 片一组分包并行。
- **单片三线**：精修期一片拆 结构/字幕/音频（或 视觉/脚本/字幕）三条只读审计线并行，修复归主线程；审计线不改项目、不切时间线，只交问题清单+精确 key。
- **流水线重叠**：当前片收尾时即派下一片 preflight。
- 子代理结论主动回传，不依赖轮询；审计与修复不并发互踩——结构冻结后再终验，冻结前的证据作废。

## 生成资产纪律（ChatCut 生成工具）

- **成本契约**：单次生成执行前必读工具显示的预估成本，超出用户已授权范围先报价再执行；一次任务预估生成调用 ≥3 次时，合并成一次提案报预估总成本让用户选全部/部分/不加，不逐条打断也不静默连发。
- **降级预案**：生成服务有高峰期全线超时的历史，关键交付不押生成成功——每个生成资产预先声明降级备选（真实录屏局部放大 / 原始截图 / 静态卡 / 已入库 MG），连续三轮失败即切降级，与「自动修复三轮上限」共用同一出口。
- **参考图先行（默认必做）**：任何 image/video 生成及风格敏感的 MG 生成前，先上传参考图——已冻结色板的样片帧、品牌截图或已确认切片的代表帧，再下生成 prompt。参考图是官方明示「显著改善结果」的第一杠杆，也是色板的像素级载体。
- **两段式视频生成**：生成 B-roll 默认先用 image 生成产出首帧静态图并经人工确认，再以该图为起点跑 video 生成。证据类画面（操作演示、数据结果、报错）禁止用生成素材替代，两段式不构成豁免。
- 生成/自动匹配的 B-roll 存在「重复且泛」的实锤：入片前查重并判断语义贴合，泛素材直接拒绝，不因生成成功就入片。

## 验证方法学

- **双证据原则**：分阶段 `read_project` 结构回读 + `view_timeline_frames` 合成像素，二者都过才算验证。
- **抽帧法**：状态切换点取 `(N,N+1)` 帧对 + 末帧 + 每状态中段代表帧；过渡另取 20%/50%/80%。多帧下载后 `ffmpeg tile=3x3` 拼 contact sheet 一次目检，省 token 省审批。
- **钳制帧双读**：时长边界帧号会被钳制到最后可见帧，双读比对一致再采信。
- **批量网格只筛查**：可疑帧单独请求精确帧并独立复验 ≥2 次；不靠反复重渲染到偶然成功。
- **写入成功≠落盘**：Zoom/FX 宣称添加成功后必须回读 fx-track 实例确认存在（曾发生批量「已加」实际 0 条）。
- **素材池≠时间线**：正确资产存在于媒体池不算证据，必须验证 active timeline instance。
- **变速换算**：`find_transcript` 词级帧未折算 `playbackRate`，精确落刀按源时间换算。
- **shell 分工**：ffmpeg/ffprobe 只做证据加工（下载、拼图、音频指标、抽源帧对照）；不绕过 ChatCut 交付压平成片。宿主支持编程式批量调用时，读写比应远大于 1。
- **502/传输错误**：无画面证据时原样重试同一验证请求，不据此改时间线（写操作报错先回读，见 `recovery.md`）。

## 导出与帧率

- 用户开口才导出。短导出授权按措辞记录为「仅指定样片」或「本任务同规格验证性短导出」；范围改变或进完整成片导出，重新确认；授权不跨任务/会话。
- **帧率以实际文件为准**：30fps 时间线可能 60fps 导出；`submit_export` 区间参数按输出帧域解释，先换算 `exportFrame = round(timelineFrame × exportFps / timelineFps)`，下载后 `ffprobe` 对账真实 fps/帧数/首尾内容。文件名写着 transition 不能替代对画面内容的检查。
- 未授权导出时如实标注「最终文件未验证」。

## 编辑决策记录

高影响决定（整句删除、重排、结构变更）保留：源区间、目标区间、口述/画面证据、理由、风险、下游依赖、状态。执行后以实际回读覆盖预计值。目的可解释可复查，不强制外部文件。

## 止损与汇报

- 自动修复最多连续三轮，每轮要有新证据和明确变化；三轮后停，保留最近可用版本，报告具体区间与人工检查点。
- 卡死可产出带证据帧的交接 md 请外部 AI 复核；回件结论仍须本地验算后分级采纳。
- 汇报：只描述可见结果，不暴露内部 key/id；每个「已完成」都能映射到 revision、帧域、准确帧和证据；`unverified` 明示原因与建议人工检查点，不静默打钩。
