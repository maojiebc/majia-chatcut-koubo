# 双画面版式系统

「录屏画面 1 + 主讲人画面 2」的双画面构图：横竖版各 4 套具名版式、语义选型决策器、小窗腾挪规则。坐标真相源 = `assets/compositions.json`（本包内置快照，画布 16:9 1920×1080 / 9:16 1080×1920，均 30fps）。

## 三态叙事骨架 → 五个执行状态

口播画面在三个叙事态间流转：**A 人物全屏**（钩子/结论/转折/强情绪）、**B 双画面**（演示/操作/数据）、**C 人物加强**（演示段落里的观点承接）。落时间线时把 B、C 各拆成两个可操作状态：

| 执行状态 | 进入信号 | 退出信号 |
| --- | --- | --- |
| **A 全屏人物** | 开场钩子、强转折、重大结论、收尾 | 开始展示必须看清的屏幕证据 |
| **B-R 右侧圆窗** | 步骤/按钮/字段/数字/连续操作，关键 UI 在左中部 | 关键 UI 移到右侧，或话语转入解释 |
| **B-L 左侧圆窗** | 关键 UI、鼠标路径或字幕保护区在右侧 | UI 重心换边 |
| **C1 专业分栏** | 「为什么」「背后的逻辑」等解释段，仍需屏幕上下文 | 回到连续操作，或结论强到应回 A |
| **C2 人物加强** | 「真正重要的是…」「所以我的结论是…」等观点句 | 进入必须看清的细节操作 |

另有 **S 纯录屏**（密集 UI/复杂表格,人物 opacity=0）与 **P 隐私覆盖**（用更大人物矩形或录屏裁切完整盖住二维码等敏感区；禁止不透明白块，禁止小圆窗只遮一角）。

### 选型决策顺序（前项可否决后项）

1. **隐私与安全**：二维码、账号、密钥必须先消失。
2. **屏幕证据**：观众必须看清什么？证据密集优先 B；关键 UI 决定人物在左还是右。
3. **人物强调**：这句话靠表情/判断/情绪成立吗？解释原因优先 C1，强观点优先 C2，屏幕已无必要证据才升 A。
4. **字幕负载**：长字幕、连续数字、英文命令占用阅读带宽，保持 B 或克制的 C1。
5. **新鲜感提醒**：约 30–45 秒无构图变化时触发**复查**——但时间不是自动切换器，连续操作仍需完整证据时有理由保持 B。

### 切换落点与节奏

- 切换只落在**气口或章节边界**，不在句中切。关键句需要放大人物时，在句前气口提前约 0.3–0.6 秒起动，让画面在关键句首个有效音节附近落稳。
- 相邻两次状态变化之间至少完成一个完整语义单元；常规停留约 4–6 秒以上（A 常见 2–6s，C2 约 4–12s，C1 约 10–25s——样片起点，非硬计时器）。
- 45–90 秒切片通常 3–5 个状态段，90–120 秒通常 4–7 个；每次人物放大都应对应更重要的一句话，每次缩小都应因为录屏证据更有价值。连续两次只换形状/换边却没有语义变化 = 装饰性抖动，合并。
- 单一构图连续超过约 25 秒按静态失败复查。

## 小窗腾挪

主讲人小窗的位置变化服务于录屏讲解，不是装饰运动：

- **避让为先**：小窗不压当前讲解的 UI、数值、光标轨迹；录屏重点区换边时小窗换到对侧（pip-right/pip-left 即此用途）。
- 同版式内的位置平移在**气口处一次到位**，不做常驻漂移动画；跨版式结构变化只在章节/主题切换处做。
- 频率约束：位置变化次数 ≤ 叙事状态数。
- 每次落位后重跑保护区检查：不压字幕、不压关键 UI、区域边界不重叠。

## 竖版补充

- 竖版不是横版缩小补黑边。录屏按页面信息密度分级裁切：全貌约 5% / 概念页约 8% / 表格与密集 UI 约 10% 或定向裁切；每次裁切保护正在讲解的标题、数值、按钮、字段与光标。
- 内容保护区：录屏、讲解人、观点条、字幕边界不重叠，相邻区域保留约 10–45px 空隙。
- 目标平台含微信视频号时，字幕与关键包装收进中央 6:7 区域（1080×1260，y≈330–1590），或单独交付重排变体。
- 最终以手机观看距离验收：把代表帧缩到约 25–30% 再检查一遍字幕与录屏文字可读性。
