## 三态叙事骨架 → 七个执行状态

口播画面在三个叙事态间流转：**A 人物全屏**（钩子/结论/转折/强情绪）、**B 双画面**（演示/操作/数据）、**C 人物加强**（演示段落里的观点承接）。落时间线时把 B、C 各拆成两个可操作状态，并显式加入 S 纯录屏与 P 隐私覆盖——这样能机器检查 owner、合法转向和隐私覆盖，不把圆形、矩形、全屏当作轮播预设：

| 执行状态 | 进入信号 | 退出信号 |
| --- | --- | --- |
| **A 全屏人物** | 开场钩子、强转折、重大结论、收尾 | 开始展示必须看清的屏幕证据 |
| **B-R 右侧圆窗** | 步骤/按钮/字段/数字/连续操作，关键 UI 在左中部 | 关键 UI 移到右侧，或话语转入解释 |
| **B-L 左侧圆窗** | 关键 UI、鼠标路径或字幕保护区在右侧 | UI 重心换边 |
| **C1 专业分栏** | 「为什么」「背后的逻辑」等解释段，仍需屏幕上下文 | 回到连续操作，或结论强到应回 A |
| **C2 人物加强** | 「真正重要的是…」「所以我的结论是…」等观点句 | 进入必须看清的细节操作 |
| **S 纯录屏** | 密集 UI、连续操作或人物遮挡关键证据；人物隐藏但 anchor 仍唯一可听 | 需要人物陪伴、解释或结论强调 |
| **P 隐私覆盖** | 检出二维码、账号、手机号、密钥或个人数据 | 敏感内容已离开画面且边界复验通过 |

P 是**独立、不透明、全区间**的 redaction owner，可与叙事状态重叠；人物大矩形与录屏裁切只能改善构图，**不能作为隐私安全保证**。redaction 必须覆盖每个稳定态、全部过渡和边界帧；部分遮挡=失败（ChatCut 端的源头平移优先处置见 `chatcut-field-notes.md` 隐私扫描 SOP）。

**双素材呼吸感**：人物和录屏是两个平等的呼吸主体，各自都应该有放大到全屏/近全屏的时刻——人物升 A、录屏升 S 交替出现，不能只让人物变而录屏永远蹲在窗里。长片（>5 分钟）每幕至少评估一次 S 态机会（关键操作、纯屏幕证据、画面自身有叙事力的段落）；A 与 S 的交替是节奏的上限刺激，不是例外。

### 选型决策顺序（前项可否决后项）

1. **隐私与安全**：二维码、账号、密钥必须先消失。
2. **屏幕证据**：观众必须看清什么？证据密集优先 B；关键 UI 决定人物在左还是右。
3. **人物强调**：这句话靠表情/判断/情绪成立吗？解释原因优先 C1，强观点优先 C2，屏幕已无必要证据才升 A。
4. **字幕负载**：长字幕、连续数字、英文命令占用阅读带宽，保持 B 或克制的 C1。
5. **新鲜感提醒**：约 30–45 秒无构图变化时触发**复查**——但时间不是自动切换器，连续操作仍需完整证据时有理由保持 B。

### 切换落点与节奏

- 切换只落在**气口或章节边界**，不在句中切。关键句需要放大人物时，在句前气口提前约 0.3–0.6 秒起动，让画面在关键句首个有效音节附近落稳。
- 相邻两次状态变化之间至少完成一个完整语义单元；常规停留约 4–6 秒以上（A 常见 2–6s，C2 约 4–12s，C1 约 10–25s——样片起点，非硬计时器）。
- 45–90 秒切片通常 3–5 个状态段，90–120 秒通常 4–7 个；每次人物放大都应对应更重要的一句话，每次缩小都应因为录屏证据更有价值。连续两次只换形状/换边却没有语义变化 = 装饰性抖动，合并。
- 单一构图连续超过约 25 秒按静态失败复查。

## 状态落地的原子契约

状态表落到时间线时，把每个稳定态写成半开区间 `[startFrame,endFrame)`，同区间的三项视为一个不可拆的提交单元：①录屏层的 rect、fit/crop 与静音状态；②人物层的可见 rect、窗形/蒙版、内部取景与主音频归属；③共享框线/装饰的状态与人物轮廓 knockout。

- **状态表最少字段**：`stateId | [start,end) | 三层参数 | inTransitionId | outTransitionId | privacy | semanticReason`。这也是 SKILL.md「状态表先行」硬闸要求请用户过目的表。
- **按区间相交审计**，不按 item 起点猜状态：每个媒体 item 必须只命中一个稳定态；同时与两个状态相交时，先在合法气口拆分 item 或把状态边界移到已有切点，再同步三层。禁止先改人物与框线、以后再补录屏。
- 每个可见的非字幕 item/FX 必须且只能绑定一个 owner（开场/稳定态/过渡/片尾）；装饰层区间不越出宿主状态区间，跨多个宿主就在边界拆实例。owner 不明的旧 MG、probe、孤儿 FX、未知 override 都阻止交付。
- 宿主没有事务 API 时也按逻辑事务执行：`validateOnly → 保存修改前值 → 写入三层 → 结构回读 → 精确帧`；任一层失败即恢复修改前值并复验，不带着半成品推进下一状态或扩散到批量。
- 把 `outroStart` 作为明确帧号写入状态表；正文所有 owner 的区间必须在 `outroStart` 之前结束，`[outroStart,timelineEnd)` 只允许收束 owner 可见（片尾孤儿框线是高频事故，验收门禁见 `operating-manual.md`）。

## 小窗腾挪

主讲人小窗的位置变化服务于录屏讲解，不是装饰运动：

- **避让为先**：小窗不压当前讲解的 UI、数值、光标轨迹；录屏重点区换边时小窗换到对侧（pip-right/pip-left 即此用途）。
- 同版式内的位置平移在**气口处一次到位**，不做常驻漂移动画；跨版式结构变化只在章节/主题切换处做。
- 频率约束：位置变化次数 ≤ 叙事状态数。
- 每次落位后重跑保护区检查：不压字幕、不压关键 UI、区域边界不重叠。

## 竖版补充

- 竖版不是横版缩小补黑边。录屏按页面信息密度分级裁切：全貌约 5% / 概念页约 8% / 表格与密集 UI 约 10% 或定向裁切；每次裁切保护正在讲解的标题、数值、按钮、字段与光标。
- 竖版有两套并行的文字体系，不得混用：**字幕体系**（手机端单行字幕，84px 基线，见 `captions-terminology.md`）和**包装体系**（顶栏/章节标签/观点条/页脚等信息舞台文字，约 32px 起步、分档到 112–126px）。包装体系的小号副文不是字幕的备选字号；不要为塞更多装饰文字缩小字号。
- 裁切按镜头任务切换，不按固定秒数：同一页面先全貌、说到具体信息切局部、需要时回结果全貌；录屏窗口保持稳定尺寸，改变的是内部画面焦点。
- 内容保护区：录屏、讲解人、观点条、字幕边界不重叠，相邻区域保留约 10–45px 空隙。
- 目标平台含微信视频号时，字幕与关键包装收进中央 6:7 区域（1080×1260，y≈330–1590），或单独交付重排变体。
- 最终以手机观看距离验收：把代表帧缩到约 25–30% 再检查一遍字幕与录屏文字可读性。
