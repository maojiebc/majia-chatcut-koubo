# 字幕与词表

中文口播字幕的语义分卡、两行上限、术语纠错与可自维护词表机制。

## 四层模型（先修上游再修下游）

1. **原始转写**：语音文字与词级时间戳，时序基础。
2. **字幕源/语言变体**：字幕项显示哪个变体。
3. **分页与气口**：什么时间显示哪张语义卡。
4. **显示样式/局部覆盖**：字体、位置与极少量 display override。

上游错误没解决时不在显示层堆补丁——显示层补丁堆积（几十上百个 override）是字幕反复返工的标志。

## 第一步永远查源绑定（P0 陷阱）

中文原音的字幕挂在 `translation:zh-Hans`/`zh-CN` **译文变体轨**上是高频事故：症状 = 凭空增词、整段漏句、错词、后半段突然变英文。任何字幕任务第一步回读字幕源绑定；发现译文轨立即切回原文源、基于**最终剪后 A-roll** 重转写重分页，禁止在译文结果上继续修。

同理：删除/重排/变速后旧时间戳全部失效，必须刷新剪后转写，不能沿用剪前时间轴。剪后重建按序：①锁定 A-roll 内容与速度 → ②刷新/重生成当前时间线转写 → ③比较媒体真实时长与转写首尾，补查 ASR 未覆盖的开头结尾 → ④先只校正文字，再单独做气口、标点和分页 → ⑤生成可见字幕后再次回读全文和时间范围。宿主无法对剪后时间线刷新转写时，不能假装旧字幕仍然准确——至少逐段重新对齐受影响区间并列为未完全验证。

## 精校逐字稿是文本唯一真相源

开工先扫素材目录找用户的人工精校逐字稿（项目文件夹常见 `*逐字稿*.txt`）：有则它是字幕文本的唯一真相源，字幕校对以它对照 ASR 词表逐词修正，**禁止在 ASR 错词上自行猜测修正**。ASR 三大高发：**数字吞位**（「500 家」→「50 家」）、**繁体混入**、**专名错写**（品牌名/产品名/人名类）。精校稿与音频不一致时以音频实际说出的内容为准；拿不准的专名列待确认，不凭常识硬猜。

## ASR 准备与缓存

- 转写前先盘点语言、说话人数、专业术语和可能出现的数字；工具支持热词/提示词/说话人分离时按当前素材启用——热词只提供候选拼写，不覆盖实际发音和上下文判断。
- 说话人标签帮助多 Take/访谈选择，不得把不确定身份写成真实姓名；笑声、叹气、掌声等事件保留为剪辑信号，字幕是否显示另行判断。
- 转写按源素材缓存；源文件未变化时复用。内容删除、重排或变速后需要的是**剪后时间线转写**，不是无条件重转每个源文件。

## 词表机制（把 `templates/terminology.template.json` 变成你的）

ASR 对品牌、专名、数字的误听是系统性的，逐次手修永远修不完。机制：

1. **维护一份词表**：`正确写法 → [常见误听...]`。开工先通读脚本/转写建立本片术语面。
2. 源层修复用 `manage_transcript` fix（格式 `FIX [段号]: 词索引 -> "正确词"`，段号纯数字；不确定词索引先用 `preview` 探针试探；空串可删词）。一次只修有证据的词。
3. **上下文优先于发音**：说话人口音导致的误听（如「500 家店」被听成「50 家店」），结合上下文判断该写什么，不盲从 ASR。
4. 疑似语义错字（如「盈利/亏损」方向性词）必须试听确认后改，不凭常识盲改。
5. 每次剪辑发现新误听就回写词表——词表是唯一维护点，别散落在对话里。

## 气口分卡

「一个可独立理解的气口 = 一张字幕事件」。事件默认排成一行；完整语义在安全宽度内确有需要时可自然换成两行，但换行不产生新的出现/消失时间点。优先在这些位置切换事件：明显吸气或停顿、语调落下、主谓宾完整、转折/因果/列举的自然边界。避免：按固定字数机械切割、拆开定语与中心词、把上句结尾和下句开头拼一张卡、2–5 字连续闪卡。

- **字符上限不是分段算法**：先按完整语义生成字幕事件，再检查阅读速度和真实像素宽度。字符数只是一道布局门禁；超限时必须回到句法边界重分，禁止在第 N 个字处直接截断。
- **三阶段决策**：①语义事件——谓词及其补语、定中短语、数量与单位、并列项内部不可拆；②时间——结合词级时间戳、停顿和阅读速度，避免过短闪卡；③布局——在实际字体、字号和安全宽度下测量，装不下才在最近的合法语义边界重分。
- **并列结构保护**：`还是 A 太高了 / 还是 B 涨价了`、`因为…所以…`、`不是…而是…` 等结构以完整分项为卡片候选；不得把 `人工｜太高了`、`原料｜涨价了` 这类谓词组拆开，也不得把下一个分项的引导词塞进上一卡。
- 经验范围：一张卡约 8–24 个汉字、0.8–3 秒（1.2–1.3x 变速下尤其避免长字串停留过短）。这是候选范围，不是硬切阈值；完整短句可少于 8 字，长句超出视觉预算时按从句或并列项重分。
- 标点：卡内自然停顿保留逗号；卡片正好在逗号处切换时删除行尾逗号（切卡本身已表达停顿）；句末句号/问号/感叹号保留。
- 分页原语：`forcePageBreak` 开新事件、`keepWithPrevious` 并入上一事件、`hidden` 藏口头词。**keepWithPrevious 禁止跨剪辑点长链连打**（会把两个气口粘成超长卡）；同一处连修 3 轮仍被引擎拆开就停手——回读结构找真断点（往往在素材切点），或回源层重建。ChatCut 分页引擎的机器路径（词级 key 获取、语义断点、源 item 边界强分页、split 后 key 前缀更换）见 `chatcut-field-notes.md` 字幕节。
- display override 低自由度：结构化审计数据必须同时保存原词 `sourceText` 与显示词 `text`，validator 由两者差异推导 override，不能靠输入方自报 `edited`；`edited` 仅保留为旧适配器的一致性断言。`text` 只校正当前词的同一词汇单元；禁止把整句写到一个短词 key 上再藏其他词（产出「文字对但只闪一帧」的伪合并）。批量 override 每批 ≤ 约 15 条，提交后立即回读（大批量会局部写入）。

## 语义事件与两行上限门禁（机器化）

样式值只是输入，viewer-facing 结果才是验收对象：

1. 应用样式/profile 后立即回读配置：`maxLines` 不得大于 2。不可放宽的发布策略在 `rules/policy.json`（最多两行、原文源、词级审计、毫秒短卡、繁体零容忍）；profile 可以保持单行，也可为完整语义启用两行，但不能把自动换行位置当成新的时间事件。
2. 全片每页 `lines∈{1,2}`；默认一行。两行页必须仍是同一完整语义事件，并对最长页、英文术语页、数字页抽帧看真实像素与逐行宽度。
3. 短卡按**时间**而非帧数判定（30/60fps 得到等价结果）：`<450ms` 一律失败；`450–800ms` 只允许词表白名单里的英文品牌/缩写完整卡（如 AI、CLI、ETL）。单字卡=0。
4. 最终回读带 `words=true`，优先保存为符合 `schemas/captions.schema.json` 的结构化 JSON，跑校验脚本：

```bash
node scripts/validate-caption-pages.mjs \
  --strict \
  --profile <你的 operating-profile.json> \
  --root <profile 配置根目录> \
  --input <captions.json 或 read_captions 输出.txt>
```

发布态使用 `--strict`：warning 也会阻断；可继承的 profile 用 `--root` 显式限定其配置根目录。结构化字幕先过 `captions.schema.json`，metadata 必须把 `projectId`、`timelineId`、`timelineRevision`、`sourceAssetId`、`sourceRevision` 五项一起绑定，并与 resolved profile 的项目、时间线成员关系、`validatedTimelineRevisions[timelineId]`、`validatedSourceRevisions[sourceAssetId]` 逐项精确一致；缺项、空值、只填一部分或已记录证据不等都失败。每个 word 继承 document 的 timeline 绑定，另外显式保存全片唯一的 `sourceWordKey`、`sourceAssetId`、`sourceRevision` 和 `sourceText`/`text`；旧 JSON 缺这些词级绑定仅能在非严格迁移模式告警通过。

`sourceText != text` 时必须有 `correction`（`approved/reason/evidenceRefs/reviewedBy/reviewedAt`）；命中词表时还要引用稳定 `termId`。数字、金额、日期、百分比、单位、否定变化，以及声明了 `risk` 或 `requiresAudioEvidence` 的词表校正，至少要有一个 `audio:` 证据。450–800ms 短卡要在 `shortCardEvidence` 中同时给 reviewer 和 `frame:`/`pixel:`/`image:` 像素证据，严格模式才放行。真实 CR/LF 无论 `lines` 写几都失败；`A / B`、`m/s` 之类正文斜杠不是换行证据。

退出码非 0 = 未完成，不得只报「已设置两行上限」。脚本检查配置、实际行数、逐行字符预算、单字卡、毫秒短卡、全文 word key 唯一、页/词安全整数区间与顺序/重叠、数值符号语义、文本一致性、繁体、override 滥用与术语黑名单；旧文本输入仅作迁移兼容。个人词表可用 `--terms <你的词表.json>` 覆盖 profile 里的 `terminologyFile`（本地个人层用法见 SKILL.md）。

字符门禁通过只说明“装得下”，不说明“分得对”。发布前另做语义边界审计：检查卡尾悬空功能词、卡首补语/趋向词、被拆开的谓词组、数量与单位、并列项和用户点名 canary。任一命中都失败，即使字符数、行数和时长全部合格。

## 起始样式基线（作者实测，填进你自己的 profile 后以你的为准）

- 横版 1920×1080：得意黑（Smiley Sans）700、38–48px，默认单行、最多两行；每行字符数只作初筛，底部安全区居中、黑底或主题 surface 圆角底板。
- 竖版 1080×1920：84px，默认单行、最多两行；`top≈1563` 需按两行高度上移，避免侵入底部安全区（72px 在手机观看距离下偏小）。
- 一行装不下时：先判断当前内容是否仍是一个完整语义事件；是则在合法短语边界自然换成两行，不是则在合法句法/气口处拆成两个时间事件。**不允许固定字数硬切，也不允许让自动换行反过来决定字幕何时出现或消失**。

## 故障恢复速查

- 字幕突然变英文 → 查源绑定（见上），切回中文源重建，不在英文结果上继续编辑。
- 修了字幕画面没变 → 只创建了变体没选择它；应用 source 后立即回读。
- 一次隐藏导致整段消失 → display key 被多实例共享；撤回最新 override，回源层修。
- 同句极短区间重复闪烁 → 查分页 key 重复；回源层重建该段，不用隐藏共享 key「消重」。
- 变速片的精确落刀 → `find_transcript` 词级帧未折算 playbackRate，按源时间换算。
